Setting up the debug server in SafiWorkshop
SafiWorkshop comes with a version of SafiServer embedded in the application for debugging and testing of Saflets (i.e. callflow applicatons). This mini-server also serves the purpose of a way of trying out the SafiServer feature set without needing to install the production standalone SafiServer.
  
1) Configure the Debug SafiServer:
This is the server that runs inside SafiWorkshop. You can configure the debug server by double clicking the Debug: node in the navigator or ...

Click here to open the Debug SafiServer Manager

You should see the following popup:



Fill in (or use the dropdown list) the Bind IP Address field with an IP address that is visable from the Asterisk server you want to use. The FastAGI and Safi Mgt can be left with the default values.

Press OK to save the changes.
