ActionStep

Directory

Description

Asterisk menu application the looks up extensions based on the name associated with the mailbox associated with the extension.

The caller is prompted to key in 3 digits representing the name of the intended destination. The application then presents the caller with a list of extensions from which they may select one. The list of names and extensions is discovered from voicemail.conf. The VM Context argument is required, and specifies the context of voicemail.conf to use. The dial-context is the context to use for dialing the users, and defaults to the vm-context if unspecified.

On exit, this ActionStep with dial the selected extension.

Example Contexts (trixbox):
Dial Context = from-did-direct
VM Context = default

Attributes

Name Default Description
Call1 Call1 The object representing the call to run Directory application on
Dial Context The context to use for dialing the found extension
Enter By First Name false ( true | false ) Allow the caller to enter the first name of a user in the directory instead of using the last name.
Enter By First Or Last Name false ( true | false ) Allow the caller to enter the first name OR last name of a user in the directory.
Name Directory Name used in the SafiWorkshop for the ActionStep
Read Extension Number false ( true | false ) In addition to the name, also read the extension number to the caller before presenting dialing options.
VM Context Required Specifies the context of voicemail.conf to use
