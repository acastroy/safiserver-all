ActionStep

SayDateTime

Description

Plays the date and time of the passed in value.

To return the current date-time; set the Date Time value to the following script:
new Date();

Attributes

Name Default Description
Call1* Call1 The object representing the call to play to
Date Time Can be a number (representing the number of seconds since 1970), text (formatted mm/dd/yyyy hh:mm:ss), or a datetime object (can be created using script)
Escape Digits # Key (or keys) user can press to skip this ActionStep
Format Format Codes
Code Description
\'filename\' the name of a soundfile to play Backslash and single quote (\') around the filename are required
A (or a) Day of week (i.e. Saturday, Sunday, Monday, etc.)
B (or b or h) Month name (i.e. January, February, March, etc.)
d (or e) Numeric day of month (i.e. first, second ... thirty-first)
Y Year
I (or l) Hour, 12 hour clock
H Hour (in 24 hour clock format: i.e. oh one, oh two ... twenty three)
k Hour (in 24 hour clock format: i.e. one, two ...twenty three)
M Minute (i.e. oh one, oh two ... fifty-nine)
m Numberic value of Month (i.e. 1st - 12th)
P (or p AM or PM) Says AM or PM
Q "today", "yesterday", or ABdY
q "" (for today), "yesterday", weekday, or ABdY
R Full 24 hour time (including minutes)
S Seconds

Example: The format: ABdY \'digits/at\' IMp with the date of 5/10/2008 9:33 PM would play "Saturday May tenth at nine thirty-three PM"
Name* SayDateTime Name used in the SafiWorkshop for the ActionStep
Timezone Acceptable values can be found in /usr/share/zoneinfo on the Asterisk server. If blank; defaults to machine default.
